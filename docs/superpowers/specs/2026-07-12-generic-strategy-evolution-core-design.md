# A股通用策略进化内核 v1 设计

## 目标

将 `D:\codex\量化构建\polymarket_quant_bot` 中已经验证的确定性候选、滚动折评估、状态指纹、影子晋级和回滚能力，适配到当前 A 股研究项目。第一阶段只服务 `strong_pullback_satellite`，不改变策略信号、A 股成交规则或每日正式配置。

## 方案选择

评估过三种方案：

1. 直接复制 Polymarket `evolution.py` 并替换市场字段。实现快，但会把钱包、订单簿和 USDC 语义带入 A 股项目，形成第二套回测规则。
2. 抽取市场无关的通用进化内核，并为强势回调策略增加适配器。复用现有 A 股回测器，后续可扩展到趋势起爆、多因子和其他卫星策略。
3. 继续扩充 `strong_pullback_evolution.py`。改动最少，但状态、指纹和影子生命周期会继续与单一策略耦合。

采用方案 2。它能保留现有交易真实性，同时减少未来策略家族重复实现进化控制器。

## 范围

### v1 包含

- 新增市场无关的 `strategy_evolution_core.py`。
- 不可变的 champion、challenger、shadow 和 rollback 状态。
- 参数、输入数据、滚动折和实验的稳定指纹。
- 相同输入、相同参数、相同随机种子产生相同候选顺序和实验标识。
- 多滚动折晋级门槛：最少有效折数、最少交易数、正收益折占比、收益改善、最大回撤、回撤恶化幅度、换手率和收益集中度。
- 影子版本只写入独立状态文件，不覆盖任何正式 YAML。
- 影子版本在后续评估不达标时回滚到前一 champion。
- 强势回调适配器继续调用 `run_strong_pullback_satellite.py` 的现有执行逻辑。
- 现有固定训练/验证/保留测试集继续作为最终人工确认门槛。

### v1 不包含

- 自动修改 Python 策略代码。
- 自动替换每日流水线当前策略。
- 券商连接、下单或账户操作。
- 将 Polymarket 钱包、盘口深度、USDC 或跟单延迟模型带入 A 股。
- 同时接入趋势起爆、多因子或其他策略家族。
- 每日自动搜索参数；进化命令保持独立运行。

## 架构

### 通用内核

`strategy_evolution_core.py` 只处理控制面，不读取行情，也不执行交易：

- `EvolutionState`：保存 schema 版本、champion、shadow 状态、历史 champion、最近运行、数据指纹和阻断原因。
- `FoldMetrics`：统一描述单个滚动折的收益、回撤、Sharpe、交易数、换手率和集中度。
- `PromotionPolicy`：定义所有硬门槛。
- `evaluate_candidate()`：比较 challenger 与 champion 的逐折表现并返回逐门槛审计结果。
- `promote_to_shadow()`：仅生成新的 shadow 状态。
- `rollback_shadow()`：恢复上一 champion 并记录原因。
- `fingerprint_payload()`：对规范化后的参数、数据证据和时间折生成稳定 SHA-256 指纹。

通用内核不得 import pandas、行情模块或任何策略执行器，以保持可测试和可复用。

### 强势回调适配器

在现有 `strong_pullback_evolution.py` 和 `run_strong_pullback_evolution.py` 上增加薄适配层：

- 负责生成 A 股交易日滚动折。
- 每个折继续调用现有强势回调回测函数。
- 把净值、交易和候选明细转换为 `FoldMetrics`。
- 收益集中度按股票贡献的已实现净收益计算；无法形成有效交易样本时标记为证据不足，不用零值代替。
- 最终保留测试集仍只比较锁定冠军与初始基准，不反向参与候选搜索。

## 状态与安全边界

- 默认命令为 `--dry-run`，只写单次运行目录。
- 只有显式 `--promote-shadow` 才能更新 `outputs/evolution_state/strong_pullback.json`。
- `--promote-shadow` 也只更新影子注册表，不能改 `configs/*.yaml`。
- 每次写状态前校验 schema、上一状态指纹和当前数据指纹，避免并发或旧数据覆盖新状态。
- 状态文件采用临时文件加原子替换；失败时保留旧状态。
- 每次运行都记录 `global_state_changed`、`selected_experiment_id` 和失败门槛。

## 晋级规则

候选必须同时满足：

- 已完成滚动折数不少于配置值，默认 3。
- 每个验证/测试窗口有效交易数不少于配置值。
- 正收益测试折占比不低于默认 `2/3`。
- 平均测试收益相对 champion 至少提高配置值。
- 任一测试折最大回撤不突破绝对下限。
- 候选回撤相对 champion 的最差恶化不超过配置值。
- 平均换手率不超过 champion 的配置倍数。
- 单一股票已实现净收益贡献占比不超过配置值。
- 所有门槛输入均为有限数值；缺失指标视为证据不足，不能晋级。

满足门槛只允许进入 shadow。正式采用仍需要现有保留测试报告和人工确认。

## 数据与未来函数控制

- 滚动折按 A 股实际交易日索引构建，不按自然日长度推断。
- 每折训练数据、验证数据和测试数据在调用策略执行器前物理切片。
- 训练期只能拟合因子权重；验证期用于候选比较；测试期只用于该折的外样本评分。
- 同一折测试期结束之后的数据不得进入指标、候选生成、流动性过滤或市场状态判断。
- 当前项目的收盘后信号、下一交易日开盘成交规则保持不变。

## 产物

单次运行继续写入 `outputs/evolution_runs/<run_id>/`，新增：

- `folds.json`：实际交易日滚动折及其指纹。
- `candidate_scores.csv`：逐候选汇总和失败门槛。
- `experiments/<experiment_id>.json`：参数、父版本、数据指纹和逐折指标。
- `evolution_state_snapshot.json`：本次运行结束时的状态快照。
- `shadow_decision.md`：中文影子晋级或回滚说明。

全局影子状态仅位于 `outputs/evolution_state/strong_pullback.json`。

## 错误处理

- 参数、指标或状态包含未知字段时立即拒绝。
- 指纹不一致、状态版本不支持或并发状态已变化时停止更新全局状态。
- 单个候选失败可记录后继续；champion 评估失败则整次运行失败。
- 折数或交易数不足时返回 `insufficient_evidence`，不得降低门槛强行晋级。
- 回滚失败不得覆盖原状态文件。

## 测试

先为通用内核编写失败测试，再实现最小代码。覆盖：

- 指纹对字典键顺序不敏感，对参数或数据变化敏感。
- 同一随机种子候选顺序可复现。
- 少于最少折数或最少交易数时拒绝晋级。
- 正收益折占比、回撤、回撤恶化、换手率和集中度边界。
- 缺失或非有限指标不能晋级。
- shadow 晋级不改变正式配置。
- rollback 恢复上一 champion。
- 状态原子写入和预期上一指纹校验。
- A 股滚动折只使用实际交易日，且策略执行器接收的数据不超过折结束日。
- 现有 `test_strong_pullback_evolution.py` 和项目完整测试保持通过。

## 验收标准

- 通用内核不依赖具体市场或策略模块。
- 强势回调策略继续使用现有 A 股回测和成交约束。
- 相同输入重复运行产生相同候选评分、实验 ID 和状态内容，时间戳字段除外。
- 未显式指定 `--promote-shadow` 时全局状态不变化。
- 即使 shadow 晋级，也不修改任何正式策略 YAML。
- 任一硬门槛失败时 challenger 不得进入 shadow。
- 保留测试未通过时输出回滚建议或执行 shadow 回滚。
- 所有新增测试、强势回调进化测试和项目完整测试通过。
