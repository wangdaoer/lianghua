# CZSC 缠论结构吸收评估（2026-07-16）

## 结论

CZSC 对现有模型有帮助，但只适合先作为日线结构影子层。首版吸收已完成笔、三笔中枢、二买卖、类三买、顺畅笔支撑压力、笔趋势与分型强弱；不替换现有横截面训练、次日开盘执行、仓位与风控系统。

## 版本边界

- 当前适配并锁定 `czsc==0.10.12`。
- GitHub `1.0.0-rc.*` 已迁移到 Rust 核心，与 `0.10.*` Python 信号接口不兼容，未经独立迁移和回归验证不得混用。
- 该依赖放在 `requirements-czsc.txt`，主环境未安装或版本不符时，每日流程输出 `unavailable` 审计结果，不静默切换语义。

## 已吸收

- `cxt_second_bs_V240524`：趋势回调后的二买/二卖影子标签。
- `cxt_overlap_V240612`：顺畅笔支撑和压力影子标签。
- `cxt_bi_trend_V230824`：笔趋势状态，只作过滤研究，不单独触发交易。
- `cxt_bi_status_V230102`：只在确认分型末根 K 线触发的表里关系。
- `cxt_fx_power_V221107`：分型强弱与中枢上下文。
- `cxt_third_buy_V230228`：类三买原始标签，并另加“当前价格位于最近三笔中枢上方”的一致性复核。
- 连续结构字段：最后完成笔力度、成交量力度、信噪比、斜率、近期笔通道位置、距三笔中枢上中下沿的距离。

## 暂不吸收

- 不接入 `CzscTrader`、`Position`、`Event`、`WeightBacktest`，避免与现有交易和回测框架重叠。
- 不把一买反转作为高收益主线。
- 不做多周期共振；当前每日统一面板只有日线。
- 不把未完成笔、UBI 或仍扩展中枢升格为生产因子。
- 不直接使用 CZSC 自带数据源或可视化层。

## 防未来函数规范

1. 每条结构结果都记在 `asof` 当日收盘，而不是分型中心 K 线、笔开始或结束端点。
2. 最早执行时点固定为下一交易日开盘。
3. 不把后来确认的分型、笔或中枢回填到历史日期。
4. 连续候选特征只读取已完成 `bi_list`；未完成结构只保留为诊断字段。
5. 离散信号可能随新 K 线演化，当前全部标记为 `research_only`、`production_feature_eligible=false`。
6. 后续若做历史训练，必须逐日截断重算或流式更新，并验证两种快照逐字段一致。

## 升格门槛

- 独立样本中每个离散标签每个滚动 fold 至少 200 次事件。
- 250 个交易日热身后，拟升格连续特征非空覆盖率至少 95%。
- 必须通过逐日截断防未来函数检查、交易成本后前向收益检验和稳定性检验。
- 只有增量收益、回撤和不同市场状态下的稳定性均通过预注册门槛，才允许进入候选排序；在此之前不影响选股和仓位。
