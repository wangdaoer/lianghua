# 外部量化框架吸收记录 2026-07-09

来源：`https://x.com/KKaWSB/status/2075032774906650746`

这条内容列出九类开源量化项目。对当前项目的价值不是“直接换框架”，而是把成熟框架里的研究纪律、策略分层和验证方法吸收到 A 股日线研究流程里。当前项目仍保持本地研究、候选筛选、观察复盘，不连接券商、不自动下单。

## 结论

第一批吸收三类思想：

1. Qlib：把因子研究做成固定流水线，保留特征、标签、IC、滚动窗口、分段表现和稳定性报告。
2. Jesse：强化零前视偏差检查，继续坚持收盘后生成信号、下一交易日开盘执行，并把这个作为所有新增策略的硬门槛。
3. Freqtrade / OctoBot：把趋势跟踪、动量、突破、回调二波启动拆成可组合的信号族，但只接入日线候选和回测，不接入 crypto 交易执行。

第二批只作为研究储备：

1. FinGPT：可用于公告、新闻、舆情风险标签，但必须先有稳定中文数据源和可审计样本，不能直接进入买入信号。
2. FinRL：可用于仓位调节或风险状态实验，不作为第一阶段选股主模型，避免样本过拟合。
3. NautilusTrader：借鉴事件驱动和研究/执行一致性思想；暂不引入引擎，因为当前还不是实盘系统。

第三批暂缓：

1. Hummingbot：偏做市和 crypto 多交易所执行，和当前 A 股日线候选模型不匹配。
2. hftbacktest：需要 tick / order book / 队列位置数据，当前日线数据无法支持。

## 可落地规则

### 1. 因子流水线纪律

新增或调整因子时必须同时输出：

- 因子定义和使用字段
- 标签定义，默认仍为“次日开盘买入，次次日开盘卖出”
- 是否使用未来数据的说明
- 滚动训练窗口
- IC 均值、IC 稳定性、分段收益和最大回撤
- 与当前主模型对比，而不是只看单次收益

### 2. 零前视偏差硬门槛

新增策略不得使用当日开盘后才知道的信息去决定当日开盘成交。允许的主流程仍是：

1. T 日收盘后计算特征、趋势状态、候选分数和风险状态。
2. T+1 日开盘按约束执行。
3. 用 T+2 日开盘或之后价格评估标签和收益。

### 3. 策略族分层

当前项目可以把候选增强拆成四个研究袖珍策略：

- `trend_momentum`：趋势跟踪和动量延续。
- `breakout_acceleration`：平台突破和加速。
- `strong_pullback`：强势股回调后的二波启动。
- `hidden_accumulation`：温和放量、上涨日成交更活跃、价格缓慢抬升但未爆量追高。

这四类只提供候选分层和加减分，不直接覆盖主模型排名。

### 4. 暂不吸收的部分

以下能力不进入当前主线：

- 自动下单、交易所连接、做市、套利执行。
- tick 级撮合、订单簿队列模拟。
- 未经验证的新闻情绪直接买入信号。
- 强化学习直接选股或直接决定满仓/空仓。

## 下一步验证

1. 对四个策略族分别产出日度候选列：`strategy_family`、`family_score`、`family_reason`。
2. 在日报里增加“策略族来源”，让每只候选知道自己来自趋势、突破、强势回调还是隐性吸筹。
3. 新增稳定性报告：按策略族统计入选后 1/3/5/10 日收益、胜率、最大不利波动。
4. 只有当某个策略族连续多个滚动窗口优于主模型，才允许提升权重。

## 当前判断

这条 X 内容值得借鉴，但不值得直接安装九个框架。对我们最有价值的是 Qlib 的研究流程、Jesse 的无未来函数纪律，以及趋势/突破/回调/吸筹的策略族分层。其余偏执行、高频、做市、强化学习和舆情模型的内容，先放进研究储备。
