# Alpha Arena 策略竞技场吸收记录

## 结论

`AmadeusGB/alpha-arena` 对本项目最有价值的是“多个决策器使用同一市场时点进行可审计比较”的思路，
不是其大模型直接生成买卖指令的实现。本项目不引入外部仓库代码、不调用大模型交易，也不改变现有
多因子排序、次日开盘执行、风险控制和人工确认边界。

- 仓库：https://github.com/AmadeusGB/alpha-arena
- 本地吸收模块：`build_strategy_arena_report.py`
- 每日接点：`run_daily_model_pipeline.py` 的 `strategy_arena` 阶段

## 已吸收能力

1. 组合决策器按联赛隔离，只在同一策略族和同一执行合同内比较。
2. 每个联赛使用所有参赛净值曲线的共同交易日，重新计算收益、回撤、Sharpe、波动率、换手和敞口。
3. 正式冠军、联赛基准和影子挑战者都有明确角色，报告不会因历史收益领先而改写正式状态。
4. 没有独立净值曲线的趋势形态、CZSC 结构和主力净量进入单独的信号观察分区，禁止混入组合排名。
5. 每日保存组合榜、同类两两差异、信号观察表、中文报告、源文件哈希和幂等历史账本。
6. 核心广度风控挑战者从竞技场启用后累计至少 60 个成熟观察日；卫星动态敞口继续沿用其独立
   20 日影子账本。两者均只允许进入人工复核，不允许自动晋级。

## 明确不吸收

- 不让大模型根据提示词直接输出买入、卖出或持有指令。
- 不使用加密货币报价、交易所账户或外部仓库中的本机路径与密钥配置。
- 不把不同策略族的历史收益做成一个总榜后选冠军。
- 不把历史全样本 Pareto 占优当作独立样本证据。
- 不因竞技场结果自动修改候选排序、目标权重、正式配置或纸面账户。

## 当前联赛

- 多因子次日开盘联赛：正式冠军 `core_rank`，影子挑战者 `core_breadth_guard`。
- 强势回调卫星联赛：联赛基准 `pullback_baseline`，影子挑战者 `pullback_dynamic`。
- 信号观察分区：趋势动量、强势回调二波、平台起涨前夜、隐性吸筹、CZSC 缠论结构、主力净量。

## 晋级边界

竞技场只负责生成可比证据。任何挑战者都必须完成预注册的新独立样本观察、成本与容量复核、
回撤和收益稳定性检查，并由人工确认后才能进入下一层影子或正式评审。当前
`automatic_promotion=false`、`cross_family_promotion_comparison=false`、
`llm_trading_enabled=false`，这些字段不得被日常参数搜索打开。
