# 项目执行入口

任何代理在本仓库进行数据刷新、回测、训练、调参、报告、配置或发布工作前，必须先阅读并遵守
[`MODEL_OPERATING_SYSTEM.md`](MODEL_OPERATING_SYSTEM.md)。其中“强制行为守则（由历史问题沉淀）”
是本项目的权威执行规范；与临时任务习惯冲突时，以更严格的规则为准。

执行时必须特别遵守以下入口约束：

1. 先检查 `git status`。主工作区存在无关修改时使用隔离 worktree，不覆盖、不回退用户改动。
2. 抓取状态、源文件最新日期、OHLC 可用日期和因子字段可用日期必须分别验证；不得用一个
   `max(date)` 或 `status=ok` 代替字段完整性证据。
3. 每个步骤按自己的必需字段处理缺失值。不得因可选字段缺失误删价格标签日期，也不得在
   必需因子缺失时静默放行信号。
4. 请求的 `asof_date` 不是事实日期。随访、样本成熟和报告截止日必须来自实际可用数据。
5. 未达到预注册总样本、分桶样本和随访门槛时，只能输出临时观察，禁止晋级、调参或宣称有效。
6. 失败运行和无效产物不得混入权威输出；修复后从首个受影响步骤重跑全部下游。
7. 完成前必须运行相关测试和全量测试，核对运行卡、关键产物内容及 `git status`，并如实报告
   数据缺口、测试数量、警告和仍未满足的门槛。
8. 项目始终是研究与人工决策辅助系统；不连接券商、不自动下单、不承诺收益。

发现新问题时，不能只修当天结果：必须修最上游责任模块、增加回归测试，并把可复用教训补入
`MODEL_OPERATING_SYSTEM.md`。同类问题第三次出现时，必须将检查前移到默认预检或 CI。
